The Curative Triad

Diagnose. Plan. Heal.

Three commands. Total order. FileMayor shows you what's wrong, proposes a treatment, and executes safely — with a rollback at every step.

Step 1 of 3

AI Health Diagnosis

FileMayor scans your folder and gives it a health score from 0–100. It surfaces exactly what's wrong: screenshots buried in Documents, 14 installer duplicates, 3 GB of forgotten cache.

Powered by Gemini 2.0 Flash. No data leaves your machine.

$ filemayor explain ~/Downloads

━━ FILEMAYOR DIAGNOSIS ━━━━━━━━━━━━━━
Workspace: ~/Downloads
Files scanned: 1,248 (4.2 GB)

Health Score: 22 / 100 — POOR

Issues detected:
  ⚠️  63 screenshots mixed with documents
  ⚠️  14 duplicate installers (1.8 GB wasted)
  ⚠️  27 stale cache folders
  ⚠️  No folder structure in place

→ Run 'filemayor cure' to generate a plan.
Step 2 of 3

AI Treatment Plan

FileMayor proposes a precise treatment plan in plain English. You review every move before a single file is touched. No surprises — full transparency before execution.

Plans can be customized with --prompt for your own filing rules.

$ filemayor cure ~/Downloads

━━ CURATIVE PLAN (Gemini 2.0 Flash) ━━
Strategy: Relocate by type + project context

  📂 Screenshots/ ← 63 files
  📂 Software/    ← 14 installers
  📂 Documents/   ← 94 PDFs & docs
  🗑️  Cache/       ← 27 folders (delete)

Total moves: 171 files
Space reclaimed: 3.2 GB

→ Run 'filemayor apply' to execute.
Step 3 of 3

Safe Execution + Rollback

Every operation is journaled. FileMayor applies the plan behind a 6-layer hardened runtime — and if you change your mind, a single command restores the original state instantly.

Rollback is instant, complete, and permanent.

$ filemayor apply

Applying curative plan…
  ✓ Moved 63 screenshots
  ✓ Moved 94 documents
  ✓ Archived 14 installers
  ✓ Cleaned 27 cache folders

━━ CURE APPLIED SUCCESSFULLY ━━━━━━━━
171 files moved · 3.2 GB freed
Journal: ~/.filemayor/journal/cure.json

Changed your mind?
$ filemayor undo   # full restore, instant

AI Integration

Drive FileMayor
from any AI.

@filemayor/mcp is a standalone Model Context Protocol server. Add one JSON snippet to Claude Desktop, Cursor, or Zed — and your AI gets 14 FileMayor tools that run with the same hardened safety as the desktop app.

scan analyze explain plan apply rollback organize clean duplicates dedupe delete_files history undo_last info
🤖
Claude Desktop
One JSON block in claude_desktop_config.json
⚡
Cursor & Zed
Drop-in MCP server, zero configuration
🔧
Claude Code Skill
Drop SKILL.md in ~/.claude/skills/ and Claude knows when to reach for FileMayor
claude_desktop_config.json
{
  "mcpServers": {
    "filemayor": {
      "command": "npx",
      "args": ["-y", "@filemayor/mcp"],
      "env": {
        "GEMINI_API_KEY": "..."
      }
    }
  }
}
100% local. The MCP server is a thin stdio bridge over the same hardened core as the desktop app. No data leaves your machine. GEMINI_API_KEY is only used for AI planning — never for file access.
